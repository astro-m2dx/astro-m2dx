# Astro M²DX - Magic MDX

This project provides an example of how magic the use of [MDX](https://mdxjs.com) in [Astro](https://astro.build) can be, to create beautiful and function-rich web sites, even for non-tech editors.

_Astro M²DX_ provides tools to setup your publishing pipeline based on **clean** MDX, using [Astro's MDX support](https://docs.astro.build/en/guides/markdown-content/).

## Clean MDX === Magic MDX

With clean we mean markdown, with only a few sprinkles of JSX and without overloading the frontmatter.

## Check out the existing plugins...

<Wrap>
  <Card title="Auto Layout" href="docs/plugin-remark-astro-auto-layout">
    `remark-astro-auto-layout` lets users define a default layout for all MDX-pages in a directory
    and its subdirectories.
  </Card>
  <Card title="Element Mapping" href="docs/plugin-vite-astro-mdx-mapping">
    Plugin `vite-astro-mdx-mapping` lets users define a mapping from standard html elements to JSX
    components on a per-directory basis.
  </Card>
  <Card title="Raw MDX Content" href="docs/plugin-remark-astro-raw-mdx">
    Plugin `remark-astro-raw-mdx` adds the raw MDX content to the frontmatter, to analyze in the
    layout, e.g. to compute reading time.
  </Card>
  <Card title="Auto Import" href="docs/plugin-remark-astro-auto-import">
    Plugin `remark-astro-auto-import` lets users define a set of components that can be used in
    MDX-files without explicitly being imported.
  </Card>
  <Card title="Sections for Headings" href="docs/plugin-remark-sectionize-headings">
    Plugin `remark-sectionize-headings` wraps headings and the following paragraphs into sections,
    e.g. to style them with CSS' `nth-child(even)`.
  </Card>
</Wrap>

## Discuss upcoming plugins...

- **remark-astro-mdast** - adds the markdown AST (`mdast`) to the frontmatter, to analyze in the layout, e.g. to generate a TOC.
- **remark-astro-hast** - adds the HTML AST (`hast`) to the frontmatter, to analyze in the layout, e.g. to ???.
- **remark-astro-import-src** - imports files relative to referencing file from string `src` attributes.
- **remark-astro-title-abstract** - extracts the title and abstract of a markdown file and adds it to the frontmatter `title` and `abstract` properties.
- **remark-astro-style-directive** - lets users apply CSS classes to the HTML elements resulting from the markdown.
